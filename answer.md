# Ответы на вопросы по React формам

## 1. В чем разница между управляемыми и неуправляемыми компонентами в формах React?

**Управляемые компоненты:**
- Значения полей хранятся в состоянии React (useState)
- Изменения обрабатываются через onChange
- React контролирует состояние формы
- Состояние синхронизировано с отображением

**Неуправляемые компоненты:**
- Значения хранятся в DOM
- Доступ к данным через ref (useRef)
- React не отслеживает изменения в реальном времени
- Данные получаются при отправке формы

## 2. Когда следует использовать управляемые компоненты, а когда - неуправляемые?

**Управляемые компоненты лучше использовать когда:**
- Нужна валидация в реальном времени
- Требуется динамическое изменение формы
- Нужна мгновенная обратная связь
- Форма зависит от других компонентов
- Требуется контроль над каждым изменением

**Неуправляемые компоненты подходят для:**
- Простых форм без валидации
- Когда не нужно отслеживать каждое изменение
- Для интеграции с не-React кодом
- Когда важна производительность (меньше ререндеров)

## 3. Какие методы используются для обработки событий в формах React?

Основные методы обработки событий:
- `onChange` - при изменении значения поля
- `onSubmit` - при отправке формы
- `onBlur` - при потере фокуса полем
- `onFocus` - при получении фокуса
- `onClick` - при клике на элементы

Пример:
```jsx
const handleChange = (event) => {
  setValue(event.target.value);
};

const handleSubmit = (event) => {
  event.preventDefault();
  // обработка данных
};
