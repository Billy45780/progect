# Ответы на вопросы для самоконтроля

## 1. Что такое Node.js и в чем его преимущества для создания серверов?

Node.js - это среда выполнения JavaScript на стороне сервера, построенная на движке V8. Преимущества:
- **Однопоточная событийная модель** - эффективная обработка множества одновременных подключений
- **NPM** - богатая экосистема пакетов
- **Единый язык** - JavaScript и на клиенте, и на сервере
- **Высокая производительность** благодаря неблокирующему I/O
- **Масштабируемость** - легко создавать распределенные системы

## 2. Как создать простой HTTP-сервер в Node.js?

```javascript
const http = require('http');

const server = http.createServer((req, res) => {
    res.writeHead(200, {'Content-Type': 'text/plain'});
    res.end('Hello World!');
});

server.listen(3000, () => {
    console.log('Server running on port 3000');
});

3. В чем разница между методами GET, POST, PUT и DELETE?
GET - получение данных (идемпотентный, безопасный)

POST - создание новых ресурсов (не идемпотентный)

PUT - полное обновление ресурса (идемпотентный)

DELETE - удаление ресурса (идемпотентный)

4. Что такое middleware и для чего оно используется?
Middleware - это промежуточные функции, которые обрабатывают запросы перед их попаданием в основные обработчики. Используются для:

Логирования

Аутентификации

Парсинга тела запроса

CORS обработки

Сжатия ответов

6. Что такое CORS и зачем нужны соответствующие заголовки?
CORS (Cross-Origin Resource Sharing) - механизм безопасности браузеров, разрешающий跨域 запросы. Заголовки нужны для:

Access-Control-Allow-Origin - разрешенные домены

Access-Control-Allow-Methods - разрешенные HTTP-методы

Access-Control-Allow-Headers - разрешенные заголовки
