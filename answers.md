1.REST архитектура основывается на нескольких ключевых принципах: единообразие интерфейса, отсутствие состояния, кешируемость, клиент-серверная архитектура, многоуровневая система и возможность выполнения кода на стороне клиента. Эти принципы обеспечивают стандартизацию взаимодействия между компонентами системы.

2.PUT предназначен для полного обновления ресурса, заменяя все его содержимое, тогда как PATCH используется для частичного обновления, изменяя только определенные поля ресурса. PUT является идемпотентной операцией, в то время как PATCH может не обладать этим свойством.

3.Наиболее распространенные HTTP коды состояния включают 200 (Успех), 201 (Создано), 400 (Неверный запрос), 401 (Не авторизован), 403 (Запрещено), 404 (Не найдено) и 500 (Внутренняя ошибка сервера). Эти коды обеспечивают стандартизированную коммуникацию о результате выполнения запроса.

4.Ресурсо-ориентированный дизайн URL предполагает организацию эндпоинтов вокруг сущностей предметной области, используя существительные вместо глаголов. Иерархическая структура URL отражает отношения между ресурсами, например /users/1/orders.

5.Пагинация в API позволяет эффективно управлять большими объемами данных, уменьшая нагрузку на сервер и сеть. Она улучшает производительность за счет ограничения количества возвращаемых записей и обеспечивает лучший пользовательский опыт при постраничной навигации.

6.Валидация данных в REST API обычно реализуется на нескольких уровнях: проверка синтаксиса запроса, валидация типов данных, проверка бизнес-логики. При ошибках возвращается код 400 с детализированной информацией о допущенных ошибках.

7.Обработка ошибок в REST API должна предоставлять последовательную структуру ответов, включающую код ошибки, человеко-читаемое сообщение и при необходимости технические детали для разработчиков. Все ошибки должны соответствовать стандартам HTTP.

8.Path parameters используются для идентификации конкретного ресурса в иерархии URL, тогда как query parameters служат для фильтрации, сортировки и пагинации коллекций ресурсов. Path parameters являются обязательной частью пути, а query parameters - опциональными параметрами запроса.
