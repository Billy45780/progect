1.Аутентификация проверяет подлинность пользователя, в то время как авторизация определяет уровень доступа к ресурсам после успешной аутентификации.

2.JWT обеспечивает масштабируемость без хранения состояния на сервере, но усложняет немедленный отзыв токенов по сравнению с сессиями, которые проще контролировать, но требуют серверного хранилища.

3.Хеширование паролей преобразует их в необратимые строки для защиты при утечках данных, а bcrypt специально разработан для медленного вычисления, что затрудняет подбор паролей.

4.Salt представляет собой случайную строку, добавляемую к паролю перед хешированием для обеспечения уникальности хешей и защиты от атак с предвычисленными таблицами.

5.Обновление JWT токенов осуществляется через связку из кратковременного access token и долговременного refresh token, что позволяет обновлять сессию без повторного ввода учетных данных.

6.Защита от brute-Force включает ограничение попыток входа, применение капчи, постепенное увеличение времени ответа и автоматическую блокировку при подозрительной активности.

7.Система ролей и разрешений реализуется через назначение пользователям ролей с соответствующими правами доступа, которые проверяются промежуточным ПО перед выполнением операций.

8.Безопасная работа с JWT требует защищенного хранения токенов, использования HTTPS, установки короткого времени жизни access token, проверки подписи и исключения конфиденциальных данных в payload.
